# Secure Notepad

A privacy-focused note-taking app built with Flutter and Firebase. Notes are encrypted end-to-end, protected by biometric/PIN authentication, and synced across devices via the cloud.

> **Current Phase:** Authentication module complete. Notes & encryption modules coming next.

---

## Features (Authentication Module)

- Email/password registration with email verification
- Google Sign-In (popup on web, native SDK on mobile)
- Password reset with 60-second resend cooldown
- Live username uniqueness check with debounced Firestore query
- Password strength meter (Weak / Fair / Good / Strong)
- Auto-poll email verification screen (5-second interval)
- Route guards — unauthenticated users are blocked from protected screens
- Full dark mode support (follows system theme)
- Bilingual error messages (Urdu + English)

## Architecture Overview

### 1. Entry Point — `main.dart`

```
main() → WidgetsFlutterBinding.ensureInitialized()
       → Firebase.initializeApp()
       → runApp(ProviderScope(child: MyApp()))
```

- `ProviderScope` makes Riverpod available app-wide.
- `MyApp` is a `ConsumerWidget` that watches `routerProvider` for navigation.

### 2. Routing — `app_router.dart`

Uses `go_router` with a reactive redirect function:

```
User opens app
  ├── Not logged in? → /splash → /onboarding → /login or /register
  ├── Logged in but email not verified? → /verify-email
  └── Logged in + verified? → /home
```

**Route guard logic:**
- Unauthenticated users can only access `/login`, `/register`, `/forgot-password`, `/onboarding`
- Authenticated users are blocked from auth screens and redirected to `/home`
- Unverified users are redirected to `/verify-email`

### 3. State Management — `auth_provider.dart`

Three Riverpod providers:

| Provider                | Type             | Purpose                              |
| ----------------------- | ---------------- | ------------------------------------ |
| `authRepositoryProvider`| `Provider`       | Single `AuthRepository` instance     |
| `authStateProvider`     | `StreamProvider` | Firebase `authStateChanges` stream   |
| `authProvider`          | `StateNotifierProvider` | `AuthNotifier` with `AuthState` |

**AuthState** has 4 statuses:
- `loading` — operation in progress
- `authenticated` — user is signed in
- `unauthenticated` — no user
- `error` — operation failed (holds error message)

### 4. Data Layer — `auth_repository.dart`

All Firebase operations are in one repository:

| Method                    | What it does                                      |
| ------------------------- | ------------------------------------------------- |
| `register()`              | createUserWithEmailAndPassword → updateDisplayName → create Firestore doc → sendEmailVerification |
| `login()`                 | signInWithEmailAndPassword                         |
| `googleSignIn()`          | Web: signInWithPopup / Mobile: GoogleSignIn SDK    |
| `logout()`                | signOut from both Firebase and Google              |
| `sendPasswordReset()`     | sendPasswordResetEmail                             |
| `resendVerification()`    | sendEmailVerification (again)                      |
| `checkUsernameAvailable()`| Firestore query on `users` collection              |
| `getUserData()`           | Fetch user document from Firestore                 |

### 5. Error Handling — `app_exception.dart`

Maps `FirebaseAuthException` codes to bilingual messages:

```
'email-already-in-use' → "Ye email pehle se registered hai..." / "This email is already registered..."
'weak-password'        → "Password bohot kamzor hai..." / "Password is too weak..."
'user-not-found'       → "Is email se koi account nahi mila..." / "No account found..."
```

All errors are displayed via SnackBars in the UI.

### 6. Theme — `app_theme.dart`

- **Primary color:** `#2EC4A9` (teal)
- **Headings:** Google Fonts Sora
- **Body text:** Google Fonts DM Sans
- **Dark mode:** Auto-detected from system, uses `ThemeMode.system`
- Styled `ElevatedButton`, `OutlinedButton`, `InputDecoration`, `SnackBar`

## Setup Instructions

### Prerequisites

- Flutter SDK 3.x (Dart 3.11+)
- Firebase project with **Authentication** and **Cloud Firestore** enabled
- FlutterFire CLI installed (`dart pub global activate flutterfire_cli`)

### Step 1 — Clone & Install

```bash
git clone <repo-url>
cd secure_notepad
flutter pub get
```

### Step 2 — Firebase Configuration

The project already includes a configured `firebase_options.dart`. If you need to reconfigure:

```bash
flutterfire configure
```

This regenerates `lib/firebase_options.dart` with your Firebase project credentials.

### Step 3 — Enable Auth Providers

In the [Firebase Console](https://console.firebase.google.com/):

1. Go to **Authentication → Sign-in method**
2. Enable **Email/Password**
3. Enable **Google** (set your support email)
4. Copy the **Web client ID** from Google Cloud Console → Credentials

### Step 4 — Firestore Rules

Deploy these security rules to allow user document creation:

```firestore
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /users/{userId} {
      allow read, write: if request.auth != null && request.auth.uid == userId;
    }
  }
}
```

### Step 5 — Run

```bash
# Web (Chrome)
flutter run -d chrome

# Android
flutter run -d android

# Windows
flutter run -d windows
```

## Key Implementation Details

### Username Uniqueness

- Debounced at 600ms (fires after user stops typing)
- Queries Firestore `users` collection where `username == input`
- Shows green checkmark (available) or red X (taken) in real-time

### Password Strength Meter

Scores 0–1 based on:
- Length >= 8 characters (+0.25)
- Length >= 12 characters (+0.15)
- Contains uppercase letter (+0.2)
- Contains number (+0.2)
- Contains special character (+0.2)

### Google Sign-In (Web vs Mobile)

- **Web:** Uses Firebase's `signInWithPopup()` — no extra client ID needed at runtime
- **Mobile:** Uses native `GoogleSignIn` SDK with lazy initialization
- `GoogleSignIn` is only constructed when the user taps the button (avoids web crash)

### Email Verification Polling

- `Timer.periodic(Duration(seconds: 5))` calls `reloadUser()` on Firebase Auth
- When `user.emailVerified == true`, auto-navigates to `/home`
- Timer is cancelled on dispose or when verification succeeds
